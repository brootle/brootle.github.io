Announcing our new API Request Builder
As part of our ongoing efforts to make cloud transcoding more flexible and simple to use, we are announcing an easier way to build complex API requests with our latest tool, the Request Builder{link to page}. We are doing this because our Custom transcoding method (/v2/start_encode) is already more powerful, customizable, and feature rich than transcoding using Profiles, we've decided to build on top of those strengths, by focusing our efforts to make this transcoding workflow more accessible to all of our users.

With the new API Request Builder, you can easily generate a JSON request using the full range of encoding settings available across the Qencode API, which makes it easier than ever to integrate into your video workflow. There is also an option to save your settings and even launch transcoding jobs directly through the UI, so you will now have all the functionality of Profiles, with more features and flexibility.

As part of our efforts to focus more on expanding the functionality of Custom transcoding (/v2/start_encode), we are also announcing that that we will be sunsetting our Profile transcoding method (/v1/start_encode) on July 30, 2021.

You can still use Profiles until July 30, 2021 and you will see no major changes other than the Profile page moving to a different part of the site. To access the Profile page, go to Projects and click the three dot menu on the right and select "Legacy Profiles". You will be taken to the Legacy Profiles page and make changes to your profiles just like before.

If you would like to learn more about this or need help with your migration, please email us at support@qencode.com
